NetClip: Remote Clipboard Viewer For Windows NT 4.0

  • Do you have more than one computer?
  • Have you ever had to "cut & paste" (Rich Text!) from one machine to another?
  • Have you ever tried to use "Clipbook Viewer"? And wondered why we even bother shipping it?

If you can say Yes to any of the above, you'll dig NetClip. With NetClip, you can not only view the contents of your other machines' clipboards, but you can Drag & Drop (or Cut & Paste) any data, in any format, to and from the other machine.

NetClip uses the Microsoft Component Object Model (COM) to share the clipboard between the two machines. It fully supports Rich Text as well as any custom data format supported by your applications.

To install, simply right click on NETCLIP.INF and choose "Install" from the menu.

To use, simply run NETCLIP.EXE on any of your machines. When it starts you'll be viewing your local clipboard. To connect to another machine, use the "Connect" toolbar button to enter the machine name. You can enter a NetBIOS name (e.g. "mymachine") or a DNS name ("mymachine.foo.com") or even an IP address. The disconnect button does what you'd think.

You can also pass a machine name on the command line, allowing you to put a shortcut on your desktop for each of your machines. (Future versions will have real shell extension support).

NetClip works great on Windows 95 as a local clipboard viewer. It requires Windows NT 4.0 to be able to view and modify the clipboard on another machine (at least until distributed COM is available on Windows 95 in late 1996).
